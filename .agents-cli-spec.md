# Agent Spec: Roving Bard

## Overview
The Roving Bard is a local assistant that monitors a video game screen in real-time to detect the current in-game location and coordinates. Based on a configuration file (`mapping.yaml`), the agent plays corresponding music tracks (from a local playlist directory) and applies smooth transitional effects (such as fade-out and fade-in) when switching between different game locations.

## Example Use Cases
- **Exploring the World:** The player moves from the "Town" area to the "Forest" area. The agent captures the screen, detects the location name "Forest", fades out the town background music, and fades in the forest background music.
- **Specific Coordinate Hotspots:** The player enters a specific coordinate zone (e.g., North/South: [10.0, 20.0], East/West: [-80.0, -60.0]) representing a secret cave. The agent transitions playback to the "Secret Cave Theme".

## Coordinate System & Extraction
- Coordinates in-game are displayed as float tuples with directional hemispheres: e.g., `19.3N, 70.9W` or `14.9S, 103.1E`.
- **Parsing Logic:**
  - **North/South (Latitude-like):** North is positive, South is negative. E.g., `19.3N` -> `19.3`, `14.9S` -> `-14.9`.
  - **East/West (Longitude-like):** East is positive, West is negative. E.g., `103.1E` -> `103.1`, `70.9W` -> `-70.9`.

## Tools & Core Pipeline
1. **Screenshot Tool:** Captures the foreground screen. For privacy, it immediately crops the image to the specified mini-map and coordinates area (using relative 0-1 float bounds).
2. **Hybrid OCR/Vision Parser:**
   - **Step A (Local OCR - Default):** Preprocesses the cropped image (converts to grayscale, applies binarization/thresholding, and upscales for clarity) and runs local Tesseract OCR (`pytesseract`).
   - **Step B (Gemini Vision - Fallback):** If Tesseract fails to detect coordinates or location text, the cropped image is sent to Gemini (multimodal model e.g., Gemini 2.5 Flash / Gemini 1.5 Flash) to parse and return the coordinates and location.
3. **Audio Player:** Uses `pygame.mixer` to play supported formats (`.mp3`, `.mp4`, `.flac`, `.abc`, `.ogg`) and perform smooth cross-fades or transitions.

## Constraints & Safety Rules
- **Privacy Protection:** The screen capture tool must immediately crop the raw image to the specified mini-map boundary. Unused parts of the desktop screen must not be written to disk, logged, or sent to the LLM.
- **Volume & Focus:** The player must not freeze the application. The playback changes must handle missing audio files gracefully by logging a warning rather than crashing.

## Success Criteria
- Accurately detects location names and parses coordinates (N/S, E/W) from simulated/actual game screens.
- Seamlessly transitions background tracks using configurable fade-out and fade-in durations.
- Gracefully handles missing tracks and invalid/unknown locations.

## Configuration Details (`mapping.yaml`)
- `minimap_bounds`:
  - `x`: Relative X position of the top-left corner of the widget (0.0 to 1.0)
  - `y`: Relative Y position of the top-left corner of the widget (0.0 to 1.0)
  - `width`: Relative width of the widget (0.0 to 1.0)
  - `height`: Relative height of the widget (0.0 to 1.0)
- `transitions`:
  - `fade_out_ms`: Duration of fade-out in milliseconds
  - `fade_in_ms`: Duration of fade-out in milliseconds
- `playlist_directory`: Relative path to the folder containing music tracks
- `mappings`:
  - A list of mappings, each specifying either a `location_name` or a `coordinate_range`:
    - `ns_min`: Minimum North/South coordinate (South is negative)
    - `ns_max`: Maximum North/South coordinate
    - `ew_min`: Minimum East/West coordinate (West is negative)
    - `ew_max`: Maximum East/West coordinate
    - `track_file`: The track file to play when within this range.
